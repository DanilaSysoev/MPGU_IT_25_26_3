## 1. Конспект занятия

### Часть 1. Философия: "Never Trust User Input" (20 мин)

*Цель: Объяснить, что пользователь — это хакер (даже если он сам об этом не знает).*

**Тезисы:**

1. **Аналогия:** Валидация — это фейс-контроль в клубе. Если вы пустили пьяного дебошира (плохие данные) внутрь, поздно проверять его паспорт на барной стойке. Проверять нужно на входе.
2. **GIGO (Garbage In, Garbage Out):** Если функция ожидает число, а пришла строка, приложение упадет (500 Error). Это DoS-уязвимость.
3. **Где валидировать?**
    * *Frontend:* Для удобства юзера (красивая рамочка). Легко обходится через Curl/Postman.
    * *Backend:* **Обязательно**. Для безопасности.

### Часть 2. Python Type Hints: Основа порядка (30 мин)

*Цель: Научить писать типизированный код.*

**Тезисы:**

1. Python — язык с динамической типизацией, но для безопасности мы пишем так, будто она статическая.
2. Синтаксис: `def process_age(age: int) -> bool:`.
3. Это не жесткое ограничение (Python не упадет), но это документация и помощь IDE.

**Практика:**

* Пишем функцию без типов. Передаем туда строку вместо числа. Получаем ошибку в рантайме.
* Добавляем типы. Видим, как IDE начинает подсказывать методы.

### Часть 3. Pydantic: Схемы данных (60 мин)

*Цель: Научиться описывать "контракты" данных.*

**Тезисы:**

1. **Что такое Pydantic?** Библиотека для парсинга и валидации данных.
2. **BaseModel:**

    ```python
    from pydantic import BaseModel, EmailStr, Field

    class UserRegistration(BaseModel):
        username: str
        email: EmailStr
        age: int = Field(ge=18) # ge = greater or equal
    ```

3. **Автоматическая конвертация:** Если передать `age="25"` (строкой), Pydantic сам сделает `int(25)`. Если передать "двадцать", он вернет понятную ошибку.

**Практика:**

* Устанавливаем: `pip install pydantic email-validator`.
* Создаем модель пользователя.
* Пытаемся "скормить" ей кривые данные (dict) и ловим `ValidationError`.

### Часть 4. Сложная валидация (Custom Validators) (30 мин)

*Цель: Реализовать бизнес-правила (пароли, логины).*

**Тезисы:**

1. Стандартных типов (int, str) мало.
2. **Декоратор `@field_validator`:** Своя логика проверки.
3. **Пример с паролем:**
    * Длина > 8.
    * Наличие цифр и букв.
    * Не совпадает с логином.

**Практика (Live Coding):**
Дописываем в модель валидатор, который проверяет сложность пароля (через регулярки или просто `if`).

### Часть 5. Знакомство с FastAPI (40 мин)

*Цель: Поднять первый API эндпоинт.*

**Тезисы:**

1. **FastAPI:** Современный веб-фреймворк, построенный на Pydantic.
2. **Связка:** Вы описываете Pydantic-модель, вставляете её в аргументы функции, и FastAPI сам:
    * Прочитает JSON из тела запроса.
    * Провалидирует его.
    * Вернет 422 Unprocessable Entity с деталями ошибки, если что-то не так.
    * Сгенерирует Swagger-документацию (`/docs`).

**Практика:**

* `pip install fastapi uvicorn`.
* Создаем `main.py`.
* Эндпоинт `POST /register`.
* Запускаем сервер и идем в Swagger UI тестировать валидацию.

## 2. Домашнее задание №4: «Фейс-контроль»

**Легенда:**
Мы начинаем разработку "Корпоративного файлового менеджера". Первый этап — регистрация пользователей. Нам не нужна пока база данных, но нам нужен **безупречный прием данных**. Вы должны гарантировать, что в систему не попадет пользователь с возрастом < 0 или с паролем "123".

**Задание:**

1. **Подготовка:**
    * Создайте структуру проекта: `src/main.py`, `src/schemas.py`.
    * Установите `fastapi`, `uvicorn`, `pydantic[email]`.

2. **Схемы данных (Pydantic):**
    * В файле `schemas.py` создайте класс `UserCreate(BaseModel)`.
    * **Поля:**
        * `username`: строка, только буквы и цифры (используйте `pattern` или регулярные выражения), длина от 4 до 20 символов.
        * `email`: валидный Email-адрес (используйте `EmailStr`).
        * `password`: строка.
        * `confirm_password`: строка (для подтверждения).
        * `age`: целое число.

3. **Продвинутая валидация (Validators):**
    * **Возраст:** Запретить регистрацию, если возраст < 18 или > 100 (используйте `Field`).
    * **Пароль (Важно):** Напишите кастомный валидатор (через `@field_validator` или `@model_validator`), который проверяет:
        * Пароль содержит минимум 1 заглавную букву.
        * Пароль содержит минимум 1 цифру.
        * Пароль содержит минимум 1 спецсимвол (`!@#$%^&*`).
        * *Подсказка:* используйте модуль `re` (Regex).
    * **Совпадение паролей:** Проверьте, что `password == confirm_password`. Это делается через `@model_validator(mode='after')`.

4. **API (FastAPI):**
    * В `main.py` создайте приложение FastAPI.
    * Сделайте эндпоинт `POST /registration`.
    * Он должен принимать на вход JSON соответствующий схеме `UserCreate`.
    * **Логика:**
        * Если данные валидны: вернуть JSON `{"msg": "User created", "user": username_value}`.
        * Если данные невалидны: FastAPI сделает всё сам (вернет 422).

5. **Тестирование:**
    * Запустите сервер.
    * Зайдите в `http://127.0.0.1:8000/docs`.
    * Попробуйте отправить валидные и невалидные данные.

6. **Деплой на сервер (Linux):**
    * Закоммитьте изменения.
    * Зайдите на ваш учебный сервер (по SSH).
    * Сделайте `git pull`.
    * **Важно:** Обновите файл `.env` на сервере, если нужно (через `nano .env`).
    * Перезапустите приложение (Uvicorn).

**Формат сдачи:**

* Ссылка на репозиторий (GitHub), содержащий код.
* **Результат работы влейте в main с коммитом "Task 4 ready."**
* Отчет, содержащий демонстрацию работы развернутого на сервере приложения.
