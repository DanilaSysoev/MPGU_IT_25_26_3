## Часть 1. Конспект лекции

### 1. Как работает Web? (Кратко)

* **Клиент-Сервер:** Браузер отправляет запрос -> Сервер (наш Django) думает и отдает HTML -> Браузер рисует картинку.
* **Триада веба:**
  * **HTML (HyperText Markup Language):** Каркас, скелет, структура. (Заголовки, параграфы, блоки).
  * **CSS (Cascading Style Sheets):** Оформление, кожа, одежда. (Цвета, отступы, шрифты).
  * **JS (JavaScript):** Мышцы, поведение. (Анимация, реакция на клики без перезагрузки). *В этом курсе почти не касаемся.*

### 2. HTML5: Базовая структура

* Тег `<html>`, `<head>` (служебная инфа, кодировка utf-8, title) и `<body>` (то, что видит пользователь).
* **Семантика:** Почему нельзя всё верстать на `<div>`?
  * `<header>` — шапка сайта.
  * `<nav>` — меню навигации.
  * `<main>` — основной контент.
  * `<section>` — смысловой блок.
  * `<footer>` — подвал.
  * *Зачем:* Для поисковиков (SEO) и читалок для слабовидящих. Это признак хорошего тона.

### 3. Знакомство с Bootstrap 5

> **Аналогия:** Писать CSS с нуля — это как шить костюм самому, покупая ткань и нитки. Использовать Bootstrap — это пойти в магазин готовой одежды и подобрать пиджак, брюки и рубашку своего размера. Быстро, качественно, стандартно.

* **Подключение:** Самый простой способ — CDN (Content Delivery Network). Копируем одну строчку `<link>` в `<head>` и одну строчку `<script>` перед закрывающим `</body>`.
* **Классы:** В Bootstrap всё управляется классами. Хочешь синюю кнопку? Пишешь `class="btn btn-primary"`.

### 4. Сетка (Grid System) — Самое важное

* Контейнер (`container`): Ограничивает ширину контента по центру экрана.
* Система 12 колонок.
* Ряд (`row`) — это этаж.
* Колонка (`col-X`) — сколько места занимает элемент.
* *Пример:* Три товара в ряд? `col-4` + `col-4` + `col-4` = 12.
* **Адаптивность:** `col-12 col-md-4`. (На мобилке — на всю ширину, на компе — треть экрана).

### 5. Компоненты (Lego для сайта)

Разбираем основные элементы, которые понадобятся для блога/магазина:

* **Navbar:** Меню, которое сворачивается в "бургер" на телефоне.
* **Card:** Универсальная карточка (картинка, заголовок, текст, кнопка "Купить").
* **Forms:** Поля ввода (`form-control`), кнопки (`btn`).

## Часть 2. Практическое задание №2

**Тема:** Верстка прототип главной страницы.  
**Срок выполнения:** 1 неделя.  
**Максимальный балл:** 8 баллов.  

### Описание задания

Вам необходимо создать статический файл `index.html`, который станет прототипом главной страницы вашего будущего веб-приложения. Вы должны использовать Bootstrap 5 для оформления. Никакого Python пока писать не нужно.

> **Совет:** Выберите тему проекта сейчас. Если не знаете, что делать — делайте **Сервис обмена книгами**.

### Техническое задание (ТЗ)

1. Создайте файл `index.html` в вашем репозитории (в корне или папке `design/`).
2. Подключите Bootstrap 5 (CSS и JS) через CDN.
3. **Структура страницы:**
    * **Шапка (Navbar):** Должна содержать название проекта (Brand), ссылки "Главная", "Категории", "Войти". Меню должно быть адаптивным (сворачиваться на узком экране).
    * **Герой (Hero Section):** Большой блок приветствия под меню. Заголовок (H1), краткое описание сервиса и кнопка призыва к действию ("Начать", "Купить", "Читать").
    * **Контент (Grid):** Сетка карточек. Минимум **3 карточки** в ряд (товары, посты, услуги). Каждая карточка должна иметь: изображение (можно заглушку `https://placehold.co/300`), заголовок, краткий текст и кнопку.
    * **Подвал (Footer):** Секция внизу страницы. Копирайт (© 2024), ссылки на соцсети (можно заглушки `#`).
4. Сделайте коммит с понятным названием: `Add main page layout with Bootstrap`.
5. Запушьте изменения в репозиторий.

## Критерии оценивания (Аддитивная система)

Максимум: **8 баллов**.

| Критерий | Баллы | Описание |
| :--- | :---: | :--- |
| **Базовая структура HTML** | **+1** | Файл `index.html` содержит корректные теги `<html>`, `<head>`, `<body>`. Кодировка UTF-8 указана. |
| **Подключение Bootstrap** | **+1** | Стили Bootstrap подключены и работают (шрифты изменились, кнопки красивые). Нет горизонтальной полосы прокрутки. |
| **Навигация (Navbar)** | **+2** | Есть меню сверху. При сжатии окна браузера меню корректно адаптируется (не ломается, появляется кнопка-бургер). |
| **Сетка и Карточки** | **+3** | Использован класс `container`. Карточки выстроены в ряд (grid system). Карточек минимум 3. Они выглядят аккуратно. |
| **Семантика и чистота** | **+1** | Использованы теги `<header>` (или `<nav>`), `<main>`, `<footer>`. Код отформатирован (есть отступы). |

### Штрафы

* **-2 балла:** Использован встроенный CSS (`<div style="background: red; width: 100%">`). Мы учимся использовать классы Bootstrap, а не писать стили руками.
* **-1 балл:** Неработающие картинки (битые ссылки) или горизонтальный скролл на странице (вылезание контента за рамки).
* **-1 балл:** "Грязный код" (отсутствие отступов/индексации, всё в одну строку).
