# Урок 3. Знакомство с Django. Архитектура MVT

**Тип занятия:** Лекция + Live Coding  
**Продолжительность:** 3 астрономических часа  
**Сложность:** Высокая (первичная настройка окружения)  

## Цели урока

1. Понять, что такое Web-фреймворк и зачем он нужен.
2. Разобраться в архитектуре MVT (Model-View-Template) на примере аналогии.
3. Настроить виртуальное окружение (`venv`) и установить Django.
4. Создать первый проект и приложение (`app`).
5. Написать первую "View" (представление) и связать её с URL.

## Часть 1. Конспект лекции

### 1. Фреймворк vs Библиотека

* **Библиотека** (например, `requests`, `pandas`): Это набор инструментов. Вы — мастер, вы берете молоток, когда он вам нужен. Вы контролируете поток выполнения программы.
* **Фреймворк** (Django): Это каркас дома. Вы не можете построить стену там, где архитектор (Django) задумал окно. Фреймворк вызывает ваш код, а не наоборот (Inversion of Control).
* **Philosophy:** Django — это "batteries included". В нем уже есть админка, аутентификация, работа с БД, защита от атак.

### 2. Архитектура Django: MVT (Model - View - Template)

> **Аналогия: Ресторан**
> Представьте, что пользователь — это **Клиент**, который пришел поесть.

1. **URL (urls.py):** Хостес/Администратор на входе. Смотрит на запрос клиента ("Хочу суп") и направляет к нужному столику (вызывает нужную View).
2. **View (views.py):** Официант. Он принимает заказ. Он знает, что клиенту нужен суп.
    * Официант идет на кухню (**Model**) за данными (ингредиентами).
    * Официант отдает ингредиенты повару-оформителю (**Template**), чтобы тот красиво разложил еду на тарелке.
    * Официант несет готовую тарелку (HTML-страницу) клиенту.
3. **Model (models.py):** Кладовщик/Кухня. Работает с Базой Данных. Знает, где лежат продукты, сколько их и как их достать.
4. **Template (templates/*.html):** Повар-оформитель. Берет данные и вставляет их в HTML-разметку.

*(Примечание: В классическом MVC Django-View — это Контроллер, а Django-Template — это View. Но мы используем терминологию Django).*

### 3. Структура проекта

* **Project (Проект):** Весь веб-сайт целиком (например, "Весь ресторан"). Содержит глобальные настройки.
* **App (Приложение):** Отдельный модуль с логикой (например, "Кухня", "Бухгалтерия", "Доставка").
  * *Правило:* Один проект может состоять из множества приложений (Users, Blog, Shop).

### 4. Основные файлы

* `manage.py`: Пульт управления. Через него мы запускаем сервер, создаем миграции и пользователей.
* `settings.py`: Главный файл конфигурации (БД, языки, подключенные приложения).
* `urls.py`: Карта сайта (маршрутизатор).

## Часть 2. Практика (Live Coding)

*В этой части мы вместе со студентами проходим путь от пустой папки до работающего локального сервера.*

### Шаг 1. Подготовка

```bash
# Создаем папку проекта
mkdir my_project
cd my_project

# Создаем виртуальное окружение (изоляция зависимостей!)
python -m venv venv

# Активация (Windows)
venv\Scripts\activate
# Активация (Mac/Linux)
source venv/bin/activate
```

### Шаг 2. Установка Django

```bash
pip install django
# Проверка
python -m django --version
```

### Шаг 3. Создание проекта

```bash
# Точка в конце важна! Чтобы не создавать вложенную папку
django-admin startproject config .
```

### Шаг 4. Запуск сервера

```bash
python manage.py runserver
# Идем в браузер: http://127.0.0.1:8000/ - видим ракету! 🚀
```

### Шаг 5. Создание приложения (App)

```bash
python manage.py startapp core
```

*Важно:* Сразу после создания идем в `settings.py` -> `INSTALLED_APPS` и добавляем туда `'core',`.

### Шаг 6. Первая View (Hello World)

1. В `core/views.py`: Создаем функцию `index(request)`, которая возвращает `HttpResponse("Hello World")`.
2. В `config/urls.py`: Импортируем view и прописываем `path('', index)`.

## Часть 3. Домашнее задание №3

**Тема:** Трансформация репозитория в Django-проект.  
**Срок выполнения:** 1 неделя.  
**Максимальный балл:** 8 баллов.  

### Описание задания

Вам необходимо взять ваш репозиторий (созданный на Уроке 1) и превратить его в рабочий веб-сервер. Вы должны удалить тестовые скрипты, установить Django, создать структуру проекта и настроить отображение вашего HTML-шаблона (сделанного на Уроке 2).

### Пошаговая инструкция (ТЗ)

1. **Подготовка окружения (Ветка `feature/django-init`):**
    * Откройте ваш репозиторий в VS Code.
    * Создайте новую ветку `feature/django-init` от ветки `main`.
    * Убедитесь, что виртуальное окружение (`venv`) активно.
    * Удалите файл `main.py` (если он остался с первого урока).
    * Файл `index.html` (с второго урока) пока просто сохраните где-то рядом, он скоро понадобится.

2. **Установка и Старт проекта:**
    * Установите Django: `pip install django`.
    * **Важно:** Находясь в корне папки проекта, выполните команду:  
        `django-admin startproject config .`  
        *(Обратите внимание на точку в конце! Она создаст файл `manage.py` прямо в корне, а не во вложенной папке).*
    * Проверьте, что файл `.gitignore` корректен (он должен скрывать `venv/`, `__pycache__`, `*.sqlite3`).

3. **Создание приложения (App):**
    * Создайте приложение, которое будет отвечать за главную страницу (назовите его `pages`, `core` или по теме вашего проекта, например `blog`):  
        `python manage.py startapp pages`
    * Откройте `config/settings.py`, найдите список `INSTALLED_APPS` и добавьте туда название вашего приложения (например, `'pages',`).

4. **Интеграция верстки (MVT в действии):**
    * Внутри папки вашего приложения (например, `pages/`) создайте папку `templates`.
    * Внутри `templates` создайте папку с именем вашего приложения (например, `pages/templates/pages/`). *Это стандарт Django для избежания конфликтов имен.*
    * Переместите ваш файл `index.html` в эту папку.
    * *Итог пути:* `pages/templates/pages/index.html`.

5. **Логика (View & URL):**
    * В файле `pages/views.py` создайте функцию:  

        ```python
        def index(request):
            return render(request, 'pages/index.html')
        ```

    * В файле `config/urls.py` импортируйте эту функцию и добавьте путь:

        ```python
        path('', index, name='home'),
        ```

6. **Фиксация зависимостей:**
    * Выполните команду `pip freeze > requirements.txt`. В корне проекта появится файл со списком библиотек.

7. **Завершение:**
    * Проверьте работу: `python manage.py runserver`. Браузер должен показать вашу верстку.
    * Сделайте коммит: `git add .` -> `git commit -m "Init Django project and add main page"`.
    * Отправьте ветку на GitHub: `git push`.
    * *(Опционально для продвинутых: сделайте Pull Request в `main` и смержите его).*

### Критерии оценивания (Аддитивная система)

Максимум: **8 баллов**.

| Критерий | Баллы | Описание |
| :--- | :---: | :--- |
| **Структура проекта (Точка)** | **+2** | Файл `manage.py` находится в корне репозитория (команда `startproject` выполнена с точкой). Нет лишних вложенных папок. |
| **Чистота (Gitignore)** | **+1** | В репозитории нет мусора (`venv`, `__pycache__`, `.db`). Файл `main.py` удален. |
| **App и Settings** | **+1** | Приложение создано (`startapp`) и, самое главное, **зарегистрировано** в `settings.py`. |
| **Шаблоны (Templates)** | **+2** | Папка `templates` создана внутри приложения. Файл `index.html` находится там. Bootstrap подключен (через CDN в HTML) и работает. |
| **Работающий сервер** | **+1** | При запуске сервера и переходе на главную страницу открывается ваш шаблон, а не стартовая ракета Django. |
| **Requirements.txt** | **+1** | В корне есть файл `requirements.txt` с версией Django. |

### Штрафы

* **-2 балла:** Ошибка `TemplateDoesNotExist` (неправильно названы папки или путь в `render`).
* **-2 балла:** Проект создан во вложенной папке (например `my_repo/my_project/manage.py`). Это усложнит деплой и работу с Docker в будущем.
* **-1 балл:** Коммит сделан сразу в `main` (если вы требовали работу через ветки) или сообщение коммита бессмысленное (например "123").
